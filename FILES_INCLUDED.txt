COMPILER PHASES:
════════════════════════════════════════════════════════════════════════════════

Phase 1: Lexical Analysis (lexer.py)
  Input:  Source code
  Output: tokens.txt
  Task:   Tokenization

Phase 2: Syntax Analysis & Parsing (parser_.py)
  Input:  Tokens
  Output: symbol_table.txt, tac.txt, parser_output.txt, errors.txt
  Task:   Parsing, semantic analysis, intermediate code generation

Phase 3: Optimization (compiler.py)
  Input:  tac.txt
  Output: opt.txt
  Task:   Duplicate instruction removal

Phase 4: Code Generation (codegen.py)
  Input:  TAC, symbol table
  Output: assembly.asm
  Task:   Assembly code synthesis

TROUBLESHOOTING:
════════════════════════════════════════════════════════════════════════════════

Problem: "input.txt not found"
Solution: Create input.txt or specify correct file path

Problem: Syntax errors reported
Solution: Check for missing semicolons, unmatched brackets, undeclared variables

Problem: No output files generated
Solution: Ensure Python version 3.8+, all files in same directory

Problem: Undefined variable errors
Solution: Declare variables before use

See USER_GUIDE.md for complete troubleshooting section
